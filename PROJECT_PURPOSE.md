# CafeQ - Project Purpose (Non-Technical Summary)

## What this project is for

CafeQ is a campus cafe ordering system that helps students order food faster and helps cafe staff manage orders smoothly during busy hours.

The core goal is simple:
- reduce waiting and crowding at the counter
- organize pickup by time slots
- give clear live order status to students
- help staff prepare and hand over orders in order

## Who uses it

- Students: browse menu, place orders, pick a slot, track order progress.
- Reception/counter staff: view incoming orders by slot, print slips, mark orders ready/collected.
- Cafe owner/admin: monitor sales, manage menu, control cafe open/close, view reports, manage staff PIN.

## Main user experience (student side)

1. Student logs in with details and email OTP.
2. Student browses available menu items.
3. Student adds items (with customizations if available).
4. Student checks live pickup slot availability.
5. Student places order and receives a token.
6. Student tracks order status from received -> preparing -> ready.
7. Student picks up order at counter with the token.

It also supports:
- canceling pending orders
- viewing order history
- getting AI help for quick menu and slot suggestions

## Staff/counter workflow

The staff portal is built for fast counter operations:
- login with staff PIN
- see live orders grouped by pickup slot
- get alerts when new orders arrive
- auto/quick print order slips
- mark each order as ready, then collected
- keep the line moving with a clear slot-by-slot view

## Owner/admin workflow

The owner/admin panel is used to run the cafe daily:
- watch daily and weekly performance
- monitor active orders and bottlenecks
- manage menu items (add, price update, sold out, remove)
- open/close the cafe for ordering
- download reports (today/week/month/all)
- print invoices/order slips
- use AI business assistant for quick insights
- change receptionist PIN and reset daily token sequence

## Why this project matters

CafeQ is not just a menu app. It is an operations system for a real cafe environment:
- students get predictable pickup and less waiting
- staff get clarity and speed during rush hours
- owners get visibility and control over business performance

In short, it turns cafe ordering into an organized queue and fulfillment flow instead of a crowded counter process.
